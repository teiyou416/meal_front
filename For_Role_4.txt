# Dashboard / Calendar Page Development Guide
# 総覧ページ / カレンダーページ開発ガイド

## English

You are responsible for the Dashboard / Calendar page.

Main file to edit:

src/pages/DashboardView.vue

This page is the main overview page of the app. It should show the selected date, calendar-related information, activities, and possibly a summary of meals for that day.

You may need to use these files:

src/stores/date.ts
src/api/activity.ts
src/api/meal.ts
src/types/index.ts

Important existing state:

useDateStore

It provides:

selectedDate
selectedMonth
setSelectedDate()
goToday()

Example usage:

const dateStore = useDateStore()
const { selectedDate } = storeToRefs(dateStore)

You can use selectedDate to request activities or meals for the current day.

Possible API functions:

getActivitiesByDate(date)
getMealsByDate(date)

Recommended development scope:

1. Edit src/pages/DashboardView.vue
2. Read selectedDate from src/stores/date.ts
3. Call activity and meal APIs when the selected date changes
4. Display daily activities and meal summary
5. Do not modify router, main.ts, or request.ts unless a new route or global API behavior is required

## 日本語

あなたは総覧ページ / カレンダーページを担当します。

主に編集するファイル:

src/pages/DashboardView.vue

このページはアプリのメイン画面です。選択された日付、その日の予定、食事の概要などを表示する想定です。

必要に応じて使うファイル:

src/stores/date.ts
src/api/activity.ts
src/api/meal.ts
src/types/index.ts

既存の状態管理:

useDateStore

使える状態と関数:

selectedDate
selectedMonth
setSelectedDate()
goToday()

selectedDate を使って、その日の予定や食事データを取得できます。

使う可能性がある API:

getActivitiesByDate(date)
getMealsByDate(date)

開発時の注意:

1. 基本的には src/pages/DashboardView.vue を編集してください
2. 日付情報は src/stores/date.ts から取得してください
3. 選択日が変わったら、予定や食事データを再取得してください
4. router、main.ts、request.ts は基本的に変更しないでください
5. 新しいページ追加や共通通信仕様の変更が必要な場合のみ、基盤ファイルを変更してください
